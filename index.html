Improving the efficiency of Selenium-based load test

Selenium automates browsers to test web applications that can run in a browser. Compared to load testing based on protocol-level HTTP requests (e.g., JMeter-based load testing), Selenium-based load testing has several advantages. However, Selenium-based load testing suffers from severe performance issues, since Selenium needs to launch a browser instance for each tested user instance. In fact, a typical test machine can only test several user instances without overloading the machine.

Therefore, I worked on an approach to tackle the performance issues of Selenium-based load testing. The experimental results show that this approach can significantly increase the number of user instances that can be tested on a test machine without overloading the test machine.

  • Automation,
  • Load tests

Empirical study on Selenium-related questions on StackOverflow

Selenium users usually ask Selenium-related questions or find answers on Stack Overflow (SO). Until Nov 2018, there are 73,617 Selenium-related questions and more than 100,000 Selenium-related answers. The percentage of Stack Overflow questions tagged with ``Selenium" has been constantly increasing (i.e., it increases six folds, from 2011 to 2018), which reaches around 0.6% in 2018. In this project, we first aim to understand the issues in Selenium and how the issues are related to each other. Second, we provide a detailed analysis of Selenium-related questions that are related to programming-languages and browsers. Third, we analyze which Selenium-related questions are difficult to answer and the factors that impact the difficulties of Selenium-related questions.

  • NLP,
  • Empirical study,
  • Linear Regression,
  • StackOverflow Data
